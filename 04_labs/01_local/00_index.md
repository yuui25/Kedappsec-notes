# 00_index.md（ローカル検証基盤：作業端末・計測・証跡の全体像）

## 目的（このローカル基盤で到達する状態）
- Web/NWの検証で必要な「観測→解釈→利用→次の一手」を、ローカルで高速に回せる。
- “攻撃”のためではなく、技術理解のために **通信・挙動・差分** を確実に観測できる（HAR/ログ/pcap等）。
- 検証環境を壊してもすぐ戻せる（設定の固定・バックアップ・巻き戻しの前提化）。

## このフォルダで扱う範囲（01_local）
- 作業端末（Attack Box）の設計：OS/ツール/隔離/保存先/更新方針
- 計測・改変ポイントの設計：Proxy/証明書/経路/HTTPSの観測
- 証跡取得の設計：何をどこに残し、どの差分を見るか（再現性より技術検証の観測）

## 方針（必ず守る）
- 手順書ではなく設計書として書く（理由・境界・到達性・観測点が主役）。
- ツールの機能紹介で終えない（「何が見えるようになるか」「その差分をどう使うか」が主）。
- 取得する証跡は目的ベースで最小限（“何の判断のために取るか”を必ず書く）。
- 検証は許可範囲で行う。実環境での過剰な負荷や破壊的検証を前提にしない。

## 設計の中核（ローカルで“技術”を上げるための3要素）
### 1) 作業端末（Attack Box）
- 目的：検証を速く回し、観測/比較/自動化を自分で管理できる状態にする。
- 主眼：ツールの豊富さではなく、以下を満たすこと。
  - 検証の“視点”を固定できる（同条件で比較できる）
  - 証跡が確実に残る（後で差分を見られる）
  - 破壊しても戻せる（バックアップ/スナップショット/設定の再現）

### 2) 計測・改変ポイント（Proxy/経路）
- 目的：Webの検証で必要な「リクエスト/レスポンス/セッション/権限伝播/キャッシュ等」を観測し、意味を解釈できる状態にする。
- 主眼：プロキシを“便利ツール”としてではなく、以下の観点で設計する。
  - 何を見たいか（認証/認可/API/入力/設定）
  - どこで差分が生じるか（CDN/WAF/SSO/MFA/環境差）
  - どの経路なら観測できるか（ブラウザ→proxy→対象、アプリ→proxy等）

### 3) 証跡取得（Capture）
- 目的：検証の結論を「感覚」ではなく、観測（har/pcap/log）に基づく説明にできる状態にする。
- 主眼：再現性のための厚い報告ではなく、技術理解のための観測点を押さえる。
  - Web：HAR/Proxyログ/アプリログで、境界や権限伝播の“差分”を見る
  - NW：pcapやサービス応答で、到達性・認証要否・権限推定に繋げる

## どのtopicsの土台になるか（代表）
- ASM/OSINT：HTTP/TLS観測、外部依存推定（DNSの次の検証に接続）
- Web：認証/セッション/認可/API/入力/設定（観測→判断の核）
- Network：到達性/サービス応答の観測（必要に応じてpcap/ログへ拡張）

## 取得する証跡（目的ベース：最小セット）
- Web（必須寄り）
  - HAR（ブラウザ視点の実通信）
  - Proxyログ（改変・繰り返し・比較の視点）
- NW（必要に応じて）
  - pcap（到達性/再送/握手/暗号化境界の理解に）
- 共通
  - 検証メモ（視点：どの端末/どの経路/どの条件で見たか）

## 巻き戻し/保全（検証速度を上げる）
- ローカル設定は“壊して学ぶ”前提で、戻し方を先に決める。
- 具体は `04_labs/05_automation/02_snapshots_reset_検証の巻き戻し.md` と連動する。
- 最低限、以下を固定する。
  - 設定のバックアップ場所
  - 証跡保存場所（後で比較できる）
  - 主要ツールの設定（proxy証明書、ブラウザ設定等）

## ガイドライン対応（ASVS / WSTG / PTES / MITRE ATT&CK）
- ASVS：
  - ローカル基盤は直接要件ではなく、ASVSで扱う認証/認可/設定/ログ等を“検証可能にする土台”。
- WSTG：
  - WSTGの各テスト観点を実行し、観測し、差分を説明できる状態にする（特にINFO/認証/認可/API/設定の前提）。
- PTES：
  - Intelligence Gathering / Vulnerability Analysis / Exploitation の検証と、成立条件の切り分けを支える環境。
- MITRE ATT&CK：
  - Reconnaissance/Discovery（観測の精度）や、Credential/Impact等の理解を“観測”で支える（分類で終わらせない）。
