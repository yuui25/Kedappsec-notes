
# Katana + Burp 認証後クロール＆ログ保存手順（SPA対応・wordlist生成用）

## ■ 目的
- SPA環境でもクロール可能にする（CeWL代替）
- 認証後ページ・APIを含めてログ取得
- Burpログを保存して自作Pythonツールでwordlist生成

---

## ■ 全体フロー
~~~~
① ブラウザ → Burp経由でログイン
② Cookie取得
③ katana → Burp経由でクロール（認証付き）
④ Burpでログ収集（Logger推奨）
⑤ JSON形式で保存
⑥ 自作Pythonツールでwordlist生成
~~~~

---

## ■ Katanaコマンド

### ● 基本（認証なし確認）
~~~~
katana -u http://<target> \
  -proxy http://<burp-ip>:8080 \
  -headless \
  -xhr \
  -jc \
  -delay 2 \
  -depth 3 \
  -c 3 \
  -rl 1
~~~~

---

### ● 認証後クロール（本番）
~~~~
katana -u http://<target> \
  -proxy http://<burp-ip>:8080 \
  -H "Cookie: token=XXXX" \
  -headless \
  -xhr \
  -jc \
  -delay 2 \
  -depth 3 \
  -c 3 \
  -rl 1
~~~~

---

## ■ 主要オプション解説（重要）

- -proxy  
  → Burpに通信を流す（ログ取得の核）

- -H "Cookie: ..."  
  → 認証状態を付与（ログイン後セッション）

- -headless  
  → ブラウザとしてJS実行（SPA対応）

- -xhr  
  → API通信（fetch/XHR）を取得（SPAで最重要）

- -jc  
  → JavaScript内のURL抽出

- -delay 2  
  → SPAの非同期処理待機（API発火待ち）

- -depth 3  
  → クロール深度制限

- -c 3  
  → 同時接続数制御（負荷軽減）

- -rl 1  
  → レート制限（安全クロール）

---

## ■ 使用禁止オプション（必ず理解）

### ● -form-fill
~~~~
-form-fill
~~~~
- フォームを検出して自動入力・送信する
- POSTリクエストが発生する
- 検索・登録・ログインなどを勝手に実行する可能性あり

👉 リスク  
- データ変更  
- 意図しない操作  
- クロールの範囲を超える  

---

### ● -automatic-form-fill
~~~~
-automatic-form-fill
~~~~
- -form-fillの強化版
- hiddenパラメータも含めて自動送信
- より広範囲のフォームに対して動作

👉 リスク  
- より攻撃的な挙動  
- 状態変更の可能性が高い  

---

### ● -fuzz
~~~~
-fuzz
~~~~
- URLやパラメータを変異させてリクエスト送信
- 例：id=1 → id=2,3,4...

👉 性質  
- 完全に攻撃フェーズ（ファジング）

👉 リスク  
- 意図しない負荷  
- アクセスログ上で攻撃と判定される  

---

### ● -payload
~~~~
-payload "' OR 1=1 --"
~~~~
- 任意のペイロードを挿入してリクエスト送信
- SQLi / XSSなどの検証用途

👉 性質  
- 脆弱性攻撃そのもの  

👉 リスク  
- 完全にペネトレ攻撃扱い  

---

## ■ 結論（使用可否）

| オプション | 使用可否 | 理由 |
|------|------|------|
| -headless | ⭕ | SPA対応 |
| -xhr | ⭕ | API取得 |
| -delay | ⭕ | 非同期対応 |
| -form-fill | ❌ | POST送信発生 |
| -automatic-form-fill | ❌ | より危険 |
| -fuzz | ❌ | 攻撃 |
| -payload | ❌ | 攻撃 |

👉 今回の目的（クロール＋ログ取得）では  
👉 **攻撃系は一切不要**

## ■ 認証（Cookie取得手順）

① ブラウザ → Burp経由でログイン  
② Burp → Proxy → HTTP history  
③ ログインリクエスト選択  
④ ヘッダ確認  

~~~~
Cookie: token=XXXX
~~~~

⑤ katanaに設定  

~~~~
-H "Cookie: token=XXXX"
~~~~

---

## ■ Burpログ取得（推奨：Logger）

### ● 手順

① Burp → Loggerタブ  
② フィルタ設定  

- in-scope only  
- 静的ファイル除外（画像・css・js）

③ katana実行 → ログ確認  

---

## ■ ログ保存方法

### ● 方法①（推奨）：JSON

① 全選択  
② 右クリック → Save items  
③ JSON形式  

~~~~
{
  "request": "GET /api/user HTTP/1.1...",
  "response": "HTTP/1.1 200 OK...",
  "url": "http://target/api/user"
}
~~~~

---

### ● 方法②：Raw

~~~~
GET /login HTTP/1.1
Host: target

POST /api/login HTTP/1.1
~~~~

👉 JSON推奨（解析しやすい）
