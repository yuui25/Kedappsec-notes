# Katana + Burp 認証後クロール＆ログ保存手順（SPA対応・wordlist生成用）

## ■ 目的
- SPA環境でもクロール可能にする（CeWLの代替）
- 認証後ページ・APIを含めてログ取得
- Burpログを保存して自作Pythonツールでwordlist生成

---

## ■ 全体フロー
~~~~
① ブラウザ → Burp経由でログイン
② Cookie取得
③ katana → Burp経由でクロール（認証付き）
④ Burpでログ収集（Logger推奨）
⑤ JSON形式で保存
⑥ 自作Pythonツールでwordlist生成
~~~~

---

## ■ Katanaコマンド

### ● 基本（認証なし確認）
~~~~
katana -u http://<target> \
  -proxy http://<burp-ip>:8080 \
  -headless \
  -xhr \
  -jc \
  -delay 2 \
  -depth 3 \
  -c 3 \
  -rl 1
~~~~

---

### ● 認証後クロール（本番）
~~~~
katana -u http://<target> \
  -proxy http://<burp-ip>:8080 \
  -H "Cookie: token=XXXX" \
  -headless \
  -xhr \
  -jc \
  -delay 2 \
  -depth 3 \
  -c 3 \
  -rl 1
~~~~

---

### ● 主要オプション解説
- -proxy  
  → Burpに通信を流す（ログ取得の核）

- -H "Cookie: ..."  
  → 認証状態を付与（ログイン後セッション）

- -headless  
  → ブラウザとしてJS実行（SPA対応）

- -xhr  
  → API通信（fetch/XHR）を取得（最重要）

- -jc  
  → JavaScript内リンク抽出

- -delay 2  
  → SPAの非同期処理待機

- -depth 3  
  → クロール深度制限

- -c 3  
  → 同時接続数制御

- -rl 1  
  → レート制限（1req/sec）

---

### ● 使用禁止（今回の目的）
~~~~
-form-fill
-automatic-form-fill
-fuzz
-payload
~~~~

→ 理由：攻撃的挙動（POST送信・パラメータ改変・ペイロード注入）

---

## ■ 認証（Cookie取得手順）

① ブラウザ → Burp経由でログイン  
② Burp → Proxy → HTTP history  
③ ログインリクエスト選択  
④ ヘッダ確認  

~~~~
Cookie: token=XXXX
~~~~

⑤ katanaに設定  

~~~~
-H "Cookie: token=XXXX"
~~~~

---

## ■ Burpログ取得（推奨：Logger）

### ● 手順

① Burp → Loggerタブ  
② フィルタ設定（重要）

- in-scope only（可能なら）
- 除外：画像・css・js

③ katana実行 → ログが増えること確認

---

## ■ ログ保存方法

### ● 方法①（推奨）：JSON

① Logger or Proxy → 全選択  
② 右クリック → Save items  
③ JSON形式で保存  

~~~~
{
  "request": "GET /api/user HTTP/1.1...",
  "response": "HTTP/1.1 200 OK...",
  "url": "http://target/api/user"
}
~~~~

---

### ● 方法②：Raw（軽量）

~~~~
GET /login HTTP/1.1
Host: target

POST /api/login HTTP/1.1
~~~~

※ JSON推奨（パースしやすい）
